## Server Actions

### `submitContactForm`

**Location**: `frontend/app/actions.ts`

**Input**:

| Field | Type | Validation |
|-------|------|------------|
| name | string | Required, min 1 char, max 200 chars |
| email | string | Required, valid email format |
| message | string | Required, min 1 char, max 5000 chars |

**Behavior**:
1. Validate all fields server-side
2. Create a `contactSubmission` document in Sanity
3. Return `{ success: true }` on success
4. Return `{ success: false, error: string }` on failure

**Response**: `{ success: boolean; error?: string }`

## Revalidation

### Sanity Webhook → On-demand ISR

**Endpoint**: Handled by `next-sanity` SanityLive integration.

**Trigger**: Any document publish/unpublish in Sanity.

**Effect**: Revalidates affected pages via tag-based revalidation.
